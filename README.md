# TubeSlide

TubeSlide is an open hardware concept for a roller carriage that slides and rotates around a vertical round tube.

The goal is to create a compact positioning system that allows an object to be moved anywhere along a vertical tube while also rotating freely around it.

This repository contains the first design concept and is currently looking for engineering feedback before Prototype #1 is built.

---

## Current Status

🟡 Concept phase

No prototype has been built yet.

The current design was developed to validate the mechanical concept and collect feedback from experienced engineers before investing in manufacturing.

---

## The Problem

I live in a small apartment where space is limited.

Instead of mounting a TV on a wall or using multiple displays, I wanted a single support that could move vertically and rotate around a structural tube, allowing the screen to be positioned wherever it is needed.

The initial application is a 55" TV (approximately 10 kg), but the mechanism could potentially be adapted for many other purposes.

---

## Concept

TubeSlide consists of:

- Two split roller assemblies connected by four structural bars, each assembly containing four rollers that guide and support the TubeSlide carriage as it travels along the outer surface of a fixed, 40 mm diameter, 3 m long vertical tube.
- A payload mounting interface attached to the TubeSlide carriage. In the current application, this interface is a VESA mount designed to support a 55" TV.

The carriage can move freely along the vertical tube.

A future version is expected to rotate smoothly around the tube by replacing the fixed-direction rollers with omni wheels.

The payload interface is application-specific and can be adapted to support different devices such as TVs, monitors, cameras, lights, projectors, sensors, or other equipment.

## Potential Applications

- TV support
- Computer monitor
- Camera mount
- Lighting equipment
- Projector
- Tablet support
- Sensor positioning
- Industrial tooling
- General vertical positioning system

## Looking for Feedback

I'm mainly looking for comments about:

- Is the mechanism mechanically sound?
- What are the biggest failure modes?
- Are omni wheels a good idea?
- Is there a better way to achieve rotation?
- How would you lock the carriage safely?
- What would you improve before Prototype #1?

Any criticism is appreciated.
